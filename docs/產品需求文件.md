# GrooveScribe 產品需求文件

## 1. 核心功能需求

### 1.1 音檔上傳

系統需提供單首音檔上傳能力。

需求：

- 支援 `audio/mpeg`、`audio/wav`、`audio/x-wav`。
- 副檔名支援 `.mp3`、`.wav`。
- 第一版每次只支援上傳一個音檔。
- 上傳完成後建立 transcription job。
- API 回傳 `job_id`，前端導向任務狀態頁。

### 1.2 背景分析任務

音訊分析必須在 background worker 執行。

需求：

- API 只負責接收檔案、建立資料紀錄、派送任務。
- Worker 依序執行 preprocessing、source separation、drum transcription、post-processing、notation generation、export。
- 每個階段要更新 job status、progress、current_stage。
- 任務失敗時需保留錯誤碼與錯誤摘要。

### 1.3 結果頁

前端需提供結果頁。

需求：

- 顯示任務狀態。
- 顯示目前處理階段。
- 完成後顯示輸出檔下載按鈕。
- 完成後顯示鼓譜預覽。
- 失敗時顯示可理解的錯誤訊息。

### 1.4 檔案下載

需求：

- 使用者可下載 MIDI。
- 使用者可下載 MusicXML。
- 使用者可下載 PDF。
- 下載 API 應驗證 job 是否 completed。
- 下載 API 不直接暴露內部檔案系統路徑。

## 2. 使用者流程

```text
首頁 / 上傳頁
→ 選擇 MP3 / WAV
→ 前端呼叫 Upload API
→ API 建立 job 並回傳 job_id
→ 前端導向 /jobs/{job_id}
→ 前端輪詢 Job Status API
→ Worker 執行 AI pipeline
→ job completed
→ 前端顯示譜面預覽與下載按鈕
→ 使用者下載 MIDI / MusicXML / PDF
```

## 3. 音檔上傳限制

MVP 建議限制：

| 項目 | MVP 限制 |
|---|---|
| 格式 | MP3、WAV |
| 單檔大小 | 100 MB 以內 |
| 音檔長度 | 10 分鐘以內 |
| 聲道 | mono / stereo 皆可，後端統一轉換 |
| 取樣率 | 任意輸入，後端標準化 |
| 同時上傳 | 單檔 |
| 即時錄音 | 不支援 |

若音檔超出限制，API 應在建立任務前回傳 4xx 錯誤。

## 4. 輸出格式

### 4.1 MIDI

MIDI 是第一版最重要的輸出，因為它可被 DAW 與譜面軟體後續編輯。

MVP MIDI 內容：

- 基本 drum events。
- General MIDI drum note mapping。
- 基本 velocity。
- 基本 tempo metadata。
- 不保證完整 humanization 或演奏手法。

### 4.2 MusicXML

MusicXML 用於樂譜交換。

MVP MusicXML 內容：

- 4/4 為預設拍號；若 tempo / beat tracking 可取得可靠結果，再寫入更準確資訊。
- 使用 percussion staff。
- 將 kick、snare、hi-hat、tom、cymbal 映射到簡化鼓譜位置。
- 以可讀草稿為目標，不追求出版級 engraving。

### 4.3 PDF

PDF 用於直接查看與列印。

MVP PDF 內容：

- 由 MusicXML 轉出。
- 包含基本標題、tempo、鼓譜內容。
- 若轉換失敗，MVP 可允許 MIDI / MusicXML 成功但 PDF 標記 failed。

### 4.4 Web Preview

網頁預覽優先使用 MusicXML 渲染。

MVP 預覽內容：

- 顯示單軌鼓譜。
- 支援基本縮放或分頁瀏覽可列入 Phase 2，不是最小必備。

## 5. 錯誤處理

### 5.1 上傳錯誤

常見錯誤：

- 檔案格式不支援。
- 檔案大小超過限制。
- 音檔 metadata 無法解析。
- 上傳中斷。

回應應包含：

- `error.code`
- `error.message`
- `error.retriable`

### 5.2 Pipeline 錯誤

常見錯誤：

- `ffmpeg` 轉檔失敗。
- Demucs source separation 失敗。
- ADTOF-pytorch transcription 失敗。
- MIDI 後處理產生空結果。
- MusicXML / PDF 產生失敗。
- Worker timeout。

Pipeline 錯誤需記錄：

- job id
- stage
- exception type
- summarized message
- input artifact path
- output artifact path, if any
- worker runtime version

## 6. 使用者可預期的準確度說明

產品頁與結果頁需要清楚說明：

- 系統輸出的是「鼓譜草稿」，不是保證完全正確的人工鼓譜。
- 乾淨鼓軌通常比完整混音歌曲準確。
- 混音中鼓聲被其他樂器遮蔽時，kick、snare、cymbal 可能漏判或誤判。
- hi-hat 開閉、ride / crash 區分、ghost note、flam、drag、roll 不保證準確。
- MIDI 和 MusicXML 建議使用者下載後再進行人工修正。

建議文案：

> GrooveScribe 會產生可編輯的 AI 鼓譜草稿。主要 groove 通常可作為練習與改譜起點，但細節仍可能需要人工修正。

## 7. 未來可擴充功能

- 線上鼓譜編輯器。
- 音訊播放器與譜面同步播放。
- 手動修正 kick、snare、hi-hat、tom、cymbal。
- 多模型比較與 fallback。
- 使用者選擇音樂風格或轉譜精度。
- 更細緻的 hi-hat 開閉辨識。
- 變速歌曲 tempo map。
- 支援 `m4a`、`flac`。
- 批次上傳。
- 使用者帳號與歷史紀錄。
- 專案分享連結。
- DAW plugin 或桌面版。
