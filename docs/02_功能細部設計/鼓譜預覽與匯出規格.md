# 鼓譜預覽與匯出規格

> Level 2 功能細部設計。本文延伸 Level 1 產品與系統架構文件，不取代既有規劃。

## 功能目標

- 將 processed MIDI 或 `drum_events.json` 轉成簡單鼓譜 MusicXML，並產生 PDF。
- 提供前端 MusicXML 譜面預覽與 MIDI / MusicXML / PDF 下載。
- 以「可用草稿」為目標，不追求專業出版級 engraving。

## 使用者情境

- 使用者在結果頁看到鼓譜預覽。
- 使用者下載 MusicXML 後可匯入 MuseScore 等譜面軟體修正。
- 使用者下載 PDF 直接列印或快速查看。

## 前端行為

- completed 後呼叫 result API，取得 `preview.musicxml_url` 與 exports。
- 使用 OpenSheetMusicDisplay 載入 MusicXML 並渲染單軌鼓譜。
- 若 MusicXML 預覽失敗，顯示「譜面預覽暫時無法載入」，但仍保留下載按鈕。
- 下載按鈕依 ExportFile 狀態顯示；PDF failed 時只停用 PDF。

## 後端行為

- Result API 回傳可下載 export metadata，不直接生成檔案。
- Download API 檢查 job 與 export 狀態，再透過 Storage Adapter 回傳檔案 stream 或 signed URL。
- 下載 API 不暴露 local path。

## Worker / Pipeline 行為

- Notation generator 讀取 `drum_events.json`，建立 percussion staff notation model。
- V1 預設 4/4；若 estimated BPM 可靠，寫入 tempo metadata。
- 鼓件映射為簡化鼓譜位置：kick、snare、hi-hat、tom、cymbal。
- 產生 `score.musicxml` 後建立 `ExportFile(type=musicxml)`。
- PDF exporter 以 MusicXML 為來源產生 `score.pdf`，建立 `ExportFile(type=pdf)`；MIDI export 使用 processed MIDI 建立 `ExportFile(type=midi)`。

## 輸入資料

- `drum_events.json` 或 `processed_drum.mid`。
- `DrumTrack.estimated_bpm`、`time_signature`、title。
- Notation config：mapping table、measure grouping、renderer。

## 輸出資料

- `score.musicxml`。
- `score.pdf`。
- Export metadata：type、content_type、storage_key、file_size、status。

## API 關聯

- `GET /api/v1/transcriptions/{job_id}` 回傳 preview 與 exports。
- `GET /api/v1/transcriptions/{job_id}/download/midi`。
- `GET /api/v1/transcriptions/{job_id}/download/musicxml`。
- `GET /api/v1/transcriptions/{job_id}/download/pdf`。

## Data Model 關聯

- `ExportFile(type=midi, status=available)`。
- `ExportFile(type=musicxml, status=available)`。
- `ExportFile(type=pdf, status=available|failed)`。
- `DrumTrack.warnings` 可加入 notation simplification warnings。

## Storage / Artifact 關聯

- 輸入：`jobs/{job_id}/midi/drum_events.json`。
- 輸出：`jobs/{job_id}/notation/score.musicxml`。
- 輸出：`jobs/{job_id}/exports/score.pdf`。
- MIDI 下載來源：`jobs/{job_id}/midi/processed_drum.mid`。

## 狀態流轉

```text
midi_post_processing completed → notation_generation → musicxml export available → pdf_export → pdf available or pdf failed → completed
```

## 錯誤情境

- `NOTATION_GENERATION_FAILED`: 無法產生 MusicXML，job 應 failed。
- `MUSICXML_INVALID`: MusicXML 不能 parse。
- `PDF_EXPORT_FAILED`: PDF renderer 失敗；MIDI / MusicXML 可用時可標記 PDF export failed。
- `EXPORT_NOT_FOUND`: download API 找不到 export。
- `EXPORT_NOT_READY`: export 尚未完成。

## 驗收標準

- MusicXML 可被 OpenSheetMusicDisplay 載入。
- MusicXML 可被 MuseScore 或相容軟體開啟。
- PDF 可下載並打開；若 PDF 失敗，MIDI / MusicXML 仍可下載且 UI 有提示。
- 下載 API 不洩漏 storage path。

## 測試案例

- 單元測試：drum event 到 notation mapping。
- 整合測試：drum_events.json → MusicXML。
- 整合測試：MusicXML → PDF。
- 前端測試：completed result 顯示下載按鈕。
- 前端測試：MusicXML 載入失敗時 fallback UI。
