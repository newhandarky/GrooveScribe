# 前端頁面與互動規格

> Level 2 功能細部設計。本文延伸 Level 1 產品與系統架構文件，不取代既有規劃。

## 功能目標

- 定義 MVP 前端頁面：上傳頁、Job 結果頁、譜面預覽與下載區。
- 提供清楚的狀態回饋與準確度提醒。
- 不實作線上修譜，只提供預覽與下載。

## 使用者情境

- 使用者進入首頁即可上傳音檔。
- 使用者等待分析時看到目前進度。
- 完成後使用者預覽鼓譜並下載 MIDI / MusicXML / PDF。
- 失敗後使用者知道可重新上傳或換較清楚音檔。

## 前端行為

- 頁面一：Upload Page，包含產品簡述、檔案選擇區、限制提示、可選 title、送出按鈕。
- 頁面二：Job Result Page，路由建議 `/jobs/{job_id}`，負責輪詢 status API。
- 狀態區顯示 stage label：等待處理、正在準備音檔、正在分離鼓軌、正在辨識鼓點、正在整理 MIDI、正在產生鼓譜、正在匯出 PDF、完成、失敗。
- 完成區顯示 OpenSheetMusicDisplay 預覽與三個下載按鈕。
- 準確度提醒固定顯示：輸出為 AI 鼓譜草稿，細節可能需人工修正。

## 後端行為

- 提供前端需要的 upload、status、result、download API。
- API response 欄位需穩定，便於 TypeScript types 定義。
- Result API 應回傳 warnings 陣列供前端顯示。

## Worker / Pipeline 行為

- Worker 透過 job status 驅動前端狀態，不直接與前端通訊。
- 每個 stage 更新 progress，避免前端長時間沒有變化。
- 完成後建立 ExportFile，前端才顯示下載。

## 輸入資料

- 使用者選取的音檔。
- 可選 title。
- `job_id` route parameter。
- Status API 與 Result API response。

## 輸出資料

- Upload request。
- Job 狀態畫面。
- 譜面預覽。
- 下載行為。

## API 關聯

- `POST /api/v1/transcriptions`。
- `GET /api/v1/transcriptions/{job_id}/status`。
- `GET /api/v1/transcriptions/{job_id}`。
- `GET /api/v1/transcriptions/{job_id}/download/midi|musicxml|pdf`。

## Data Model 關聯

- 前端不直接讀 database。
- 前端 type 應對應 API response 中的 `JobStatus`、`stage`、`ExportFile.type`、`DrumTrack.warnings`。

## Storage / Artifact 關聯

- 前端不接觸 storage key。
- MusicXML preview 使用 API URL 載入。
- 下載按鈕使用 download API URL。

## 狀態流轉

```text
Upload idle → selecting file → uploading → upload success → queued/processing polling → completed result → preview/download
Upload idle → uploading → upload error
queued/processing polling → failed
```

## 錯誤情境

- 前端預檢失敗：顯示格式或大小限制。
- Upload API 失敗：顯示 `error.message`。
- Status API 暫時失敗：顯示暫時無法更新，不停止輪詢。
- Job failed：停止輪詢並顯示錯誤。
- MusicXML preview 失敗：保留下載按鈕。

## 驗收標準

- 使用者可從瀏覽器完成上傳並進入結果頁。
- 結果頁可顯示 queued / processing / completed / failed。
- 完成後可看到譜面預覽與下載按鈕。
- 失敗時有清楚提示，不出現技術 stack trace。
- 前端不依賴 local storage path。

## 測試案例

- Component test：上傳區只接受單檔。
- Component test：超過 100 MB 顯示錯誤。
- API mock test：202 後導向 job page。
- API mock test：processing 狀態持續輪詢。
- API mock test：completed 顯示 downloads。
- API mock test：failed 顯示錯誤訊息。
