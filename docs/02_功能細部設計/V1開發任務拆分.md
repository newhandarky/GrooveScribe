# V1 開發任務拆分

> Level 2 功能細部設計。本文延伸 Level 1 產品與系統架構文件，不取代 `docs/03_工程實作任務/V1_Ticket總表.md`。

## 功能目標

- 將 Level 1 與 Level 2 規格拆成完整 local-first V1 的可執行工程方向。
- 明確定義前端、後端、local job manager、AI pipeline、SQLite、local storage、測試與本機啟動的交付順序。
- 確保每個方向都有完成標準，不以「能跑通一次」作為完整 V1。

## 使用者情境

- 產品開發團隊可按階段交付，先穩定本機 runtime 與 pipeline，再建立 SQLite / local job manager / UI。
- 測試者可在每個 phase 結束時驗收具體成果。
- 使用者最終能在本機啟動 GrooveScribe 並透過 localhost 操作完整流程。

## 前端行為

- FE-01：建立 local web app shell。
- FE-02：建立上傳頁與檔案預檢。
- FE-03：串接 upload API 與導向結果頁。
- FE-04：建立 typed API client。
- FE-05：建立 job status polling。
- FE-06：建立 completed / failed / interrupted / partial success UI。
- FE-07：整合 MusicXML preview 與下載按鈕。
- FE-08：建立 runtime preflight / diagnostics UI。

## 後端行為

- BE-01：建立 FastAPI local backend 與 health / runtime API。
- BE-02：將 V1 預設 database 改為 SQLite。
- BE-03：建立 SQLite migration / upgrade 流程。
- BE-04：建立 StorageAdapter 與 LocalStorageAdapter。
- BE-05：實作 upload API。
- BE-06：實作 status / result / download API。
- BE-07：實作錯誤 response middleware 與 error catalog。
- BE-08：不把 Redis / Celery / PostgreSQL 作為 V1 必要依賴。

## Local Job Manager / Pipeline 行為

- JM-01：建立 local job manager interface。
- JM-02：實作 single-process queue 與 concurrency control。
- JM-03：實作 job state updater。
- JM-04：實作 stale / interrupted job recovery。
- AI-01：建立 ffmpeg preprocessing adapter。
- AI-02：建立 Demucs SourceSeparator adapter。
- AI-03：建立 ADTOF DrumTranscriber adapter。
- AI-04：建立 MIDI post-processing module。
- AI-05：建立 MusicXML / PDF export module。
- AI-06：建立正式 PipelineService 串接以上 stages。

## 輸入資料

- 測試音檔：乾淨鼓軌、授權真實鼓聲、一般完整歌曲、鼓聲不明顯歌曲、損壞音檔、超短音檔。
- Level 1 / Level 2 文件。
- API payload 與 artifact key convention。
- runtime preflight 結果。

## 輸出資料

- 可啟動的 Local Web App。
- 可上傳與查詢 job 的本機 API。
- SQLite metadata。
- local filesystem artifacts。
- MIDI / MusicXML / PDF。
- pipeline logs / stage reports。
- 單元、整合、pipeline、frontend、browser smoke 與人工評估測試。

## API 關聯

- API tickets 依序完成 runtime、upload、status、result、download。
- 前端 tickets 不等待完整 AI pipeline，可先用 mock API。
- Local job manager tickets 可先用 mock adapter 產生假 artifacts，再替換成真模型。

## Data Model 關聯

- DB-01：建立 SQLite schema 與 enum。
- DB-02：建立狀態轉移 service。
- DB-03：建立 ExportFile 查詢與下載檢查。
- DB-04：建立 runtime snapshot / app settings metadata。

## Storage / Artifact 關聯

- ST-01：定義 ArtifactRef 與 artifact types。
- ST-02：實作 local key naming 與 sanitizer。
- ST-03：串接 upload、pipeline、download。
- ST-04：建立 artifacts 清理、備份與重置規格。
- ST-05：保留 S3 adapter interface 作 future optional，不列入 V1 必要交付。

## 狀態流轉

建議開發順序：

```text
Phase 1 local runtime baseline
→ Phase 2 SQLite + local storage
→ Phase 3 local job manager
→ Phase 4 production pipeline service
→ Phase 5 local web app
→ Phase 6 preview/export/artifact UX
→ Phase 7 quality/diagnostics/acceptance
→ Phase 8 local startup/release checklist
```

## 錯誤情境

- 模型安裝阻塞：runtime preflight 顯示缺失，mock mode 可驗證 UI / job flow。
- PDF renderer 不穩：MusicXML 作為核心譜面輸出，PDF 顯示 partial success / failed。
- CPU 執行太慢：UI 顯示 progress、elapsed time 與 stage。
- Storage 權限問題：LocalStorageAdapter 測試與 app data 初始化需先完成。
- Service restart：stale processing job 不能永久卡住。

## 驗收標準

- Phase 1：本機 runtime check 能清楚回報 mock-ready / true-ai-ready。
- Phase 2：SQLite 與 local storage 可保存 job metadata 與 artifacts。
- Phase 3：local job manager 可排入、執行、失敗、恢復 job。
- Phase 4：正式 pipeline service 可產生 MIDI / MusicXML / PDF 與 stage reports。
- Phase 5：前端可上傳、等待、顯示完成或失敗。
- Phase 6：結果頁可預覽 MusicXML，下載 MIDI / MusicXML / PDF，處理 PDF fallback。
- Phase 7：主要測試、browser smoke 與人工評估通過。
- Phase 8：本機啟動流程與 V1 release checklist 完成。

## 測試案例

- TEST-01：validators 與 data model unit tests。
- TEST-02：SQLite migration / repository tests。
- TEST-03：storage adapter tests。
- TEST-04：API integration tests。
- TEST-05：local job manager orchestration tests with mock adapters。
- TEST-06：slow pipeline tests with fixture audio。
- TEST-07：frontend API mock tests。
- TEST-08：localhost browser smoke test。
- TEST-09：manual eval。

## 非 V1 主線

- 不先做 cloud sync。
- 不先做 SaaS 帳號、付款、多人協作。
- 不先做 Tauri / Electron shell。
- 不把 Redis / Celery 作為必要 queue。
- 不把 PostgreSQL 作為必要 database。
- 不把 S3 作為必要 storage。
- 不先做線上修譜。
- 不先做多模型 fallback，只保留 interface。
