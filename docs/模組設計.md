# GrooveScribe 模組設計

## 1. Upload Module

責任：

- 接收使用者上傳音檔。
- 驗證副檔名、MIME type、大小限制。
- 建立 `AudioFile` 與 `TranscriptionJob`。
- 將原始檔案保存到 storage。
- 推送 job 到 queue。

輸入：

- multipart form-data：`file`
- optional metadata：`title`、`source_type`

輸出：

- `job_id`
- initial job status
- status URL

可替換性：

- 上傳來源未來可從 browser upload 擴充成 signed URL direct upload。
- Upload module 不應依賴 local filesystem；需依賴 storage interface。

## 2. Audio Preprocessing Module

責任：

- 使用 `ffmpeg` 將輸入音訊標準化。
- 讀取並寫入基本音訊 metadata。
- 偵測無效或損壞音檔。

輸入：

- original audio artifact ref

輸出：

- normalized WAV artifact ref
- duration
- sample rate
- channel count
- preprocessing log

可替換性：

- `ffmpeg` adapter 應包在 service 裡。
- 未來可加入 loudness normalization、silence trimming、channel conversion 策略。

## 3. Source Separation Module

責任：

- 從完整歌曲分離出 `drums stem`。
- 保留分離後的 drums.wav。
- 可選擇保留 bass、vocals、other stems 作 debug，但 MVP 只需要 drums。

輸入：

- normalized WAV artifact ref

輸出：

- drums WAV artifact ref
- source separation metadata
- model name / version

可替換性：

- 透過 `SourceSeparator` interface 實作。
- MVP adapter：Demucs。
- 未來 adapter：Demucs fork、MDX-Net、其他 source separation model。

## 4. Drum Transcription Module

責任：

- 將 drums stem 轉成 raw drum MIDI。
- 保留模型原始輸出，供後處理與 debug 使用。

輸入：

- drums WAV artifact ref

輸出：

- raw MIDI artifact ref
- transcription metadata
- model confidence, if available

可替換性：

- 透過 `DrumTranscriber` interface 實作。
- MVP adapter：ADTOF-pytorch。
- 未來 adapter：Omnizart、custom model、ensemble transcriber。

## 5. MIDI Post-processing Module

責任：

- 解析 raw MIDI。
- 進行 tempo 對齊。
- 將事件量化到節拍格線。
- 套用 drum note mapping。
- 移除明顯重複或過密事件。
- 產出 processed MIDI 與 normalized event model。

輸入：

- raw MIDI artifact ref
- audio metadata
- optional beat / tempo metadata

輸出：

- processed MIDI artifact ref
- drum event JSON artifact ref
- post-processing report

可替換性：

- Quantization strategy 可替換。
- Drum mapping table 可設定。
- Tempo detection 可由 MVP 的簡化規則替換成更完整的 beat tracking。

## 6. Notation Generation Module

責任：

- 將 processed MIDI 或 drum event JSON 轉成 notation model。
- 產生 MusicXML。
- 簡化休止符、連桿、節奏群組。
- 使用 percussion staff 表示鼓譜。

輸入：

- processed MIDI artifact ref 或 drum event JSON artifact ref
- title / tempo / time signature metadata

輸出：

- MusicXML artifact ref
- notation generation report

可替換性：

- MVP 可用 music21 或自訂 MusicXML generator。
- 未來如需更高排版控制，可改成自訂 engraving model 或改接專業譜面服務。

## 7. Preview Module

責任：

- 前端載入 MusicXML。
- 在瀏覽器顯示鼓譜預覽。
- 顯示譜面產生狀態與 fallback 訊息。

輸入：

- MusicXML download URL 或 API response

輸出：

- rendered score preview

可替換性：

- MVP：OpenSheetMusicDisplay。
- 未來若要支援逐音符編輯，可改用 VexFlow 或自研 canvas / SVG renderer。

## 8. Export Module

責任：

- 管理可下載 artifacts。
- 產生 PDF。
- 建立 `ExportFile` records。
- 提供安全下載 URL。

輸入：

- processed MIDI
- MusicXML
- PDF export request

輸出：

- MIDI download artifact
- MusicXML download artifact
- PDF download artifact
- export metadata

可替換性：

- PDF renderer 可由 MuseScore CLI 替換成其他 renderer。
- Storage download 可由 API proxy 改成 signed URL。

## 9. Job Queue Module

責任：

- 將 transcription job 推入背景佇列。
- 控制 retry、timeout、concurrency。
- 讓 worker 更新 job 狀態。

輸入：

- job id
- pipeline config id

輸出：

- queued task id
- job status updates

可替換性：

- MVP：Celery + Redis。
- 未來可替換為 RQ、Dramatiq、Cloud Tasks、SQS、RabbitMQ。
- 業務邏輯不應直接依賴 Celery task object。

## 10. 模組間 artifact 約定

建議 artifact 命名：

| 階段 | Artifact Type | 範例 |
|---|---|---|
| upload | original_audio | `original.mp3` |
| preprocessing | normalized_audio | `normalized.wav` |
| source separation | drums_stem | `drums.wav` |
| transcription | raw_midi | `raw_drum.mid` |
| post-processing | processed_midi | `processed_drum.mid` |
| post-processing | drum_events | `drum_events.json` |
| notation | musicxml | `score.musicxml` |
| export | pdf | `score.pdf` |

每個 artifact 都應寫入：

- job id
- type
- storage key
- content type
- file size
- created at
- producing stage
