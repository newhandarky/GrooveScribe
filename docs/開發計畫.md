# GrooveScribe V1 開發計畫

## 1. V1 開發階段拆分

V1 目標是完整 local-first 產品，不是只追求最短路徑的 MVP。建議分成七個階段，每個階段都要產生可驗證、可累積到產品完整度的成果：

1. Local runtime baseline。
2. Local data、storage 與 job foundation。
3. Production pipeline integration。
4. Local Web App。
5. Score preview、export 與 artifact management。
6. 品質、可觀測性與可恢復性。
7. Packaging、啟動流程與 V1 驗收。

## 2. Phase 1：Local Runtime Baseline

目標：讓本機 AI runtime 可重現，並保留 mock / true runtime 兩條驗證路徑。

工作項目：

- 鎖定 Python runtime 與 `.venv-ai` 建立流程。
- 安裝並驗證 `ffmpeg`。
- 安裝並驗證 Demucs。
- 安裝並驗證 ADTOF-pytorch。
- 驗證 MIDI post-processing、MusicXML generation、MuseScore PDF export。
- 準備 generated fixtures 與授權真實鼓聲 fixtures。
- 建立 runtime check script，明確回報 mock-ready / true-ai-ready。

完成標準：

- 工程師可依文件重建本機 AI runtime。
- mock pipeline 可在 fixture 上穩定跑完。
- true AI pipeline 至少能在短 fixture 上產出 MIDI、MusicXML 與 PDF。
- runtime 缺失會顯示可操作的診斷訊息。

## 3. Phase 2：Local Data、Storage 與 Job Foundation

目標：建立 V1 本機資料與任務基礎，不依賴雲端服務。

工作項目：

- 將 V1 預設 database 定為 SQLite。
- 建立 schema migration / local DB upgrade 策略。
- 建立 LocalStorageAdapter 與 artifacts 目錄規格。
- 建立本機 job manager interface。
- 實作 queued / processing / completed / failed 狀態轉移。
- 定義 crash / restart 後 job recovery 或 stale job 標示策略。
- 保留 Redis / Celery / PostgreSQL 作為 optional server mode，不列為 V1 主線。

完成標準：

- 上傳檔案、job metadata、export metadata 可寫入 SQLite。
- artifacts 只寫入本機 workspace / app data 目錄，不暴露任意路徑。
- 本機 job manager 可排入 job、執行 job、記錄 progress、回報失敗。

## 4. Phase 3：Production Pipeline Integration

目標：把 Phase 1 pipeline 封裝成 V1 job manager 可呼叫的正式 pipeline service。

工作項目：

- 建立 `PipelineService.run(job_id, config)`。
- 從 local storage 讀取 original audio 到 temp workspace。
- 串接 ffmpeg、Demucs、ADTOF、MIDI post-processing、MusicXML、PDF。
- 每個 stage 寫入 progress、stage report、artifact metadata。
- 建立 pipeline error mapping 與使用者可理解錯誤。
- 記錄 pipeline version、model version、runtime version。
- 保留 raw MIDI、processed MIDI、drum events、MusicXML、PDF、pipeline log。

完成標準：

- 本機 job manager 可呼叫正式 pipeline service。
- completed job 可下載 MIDI / MusicXML / PDF。
- failed job 有穩定 error code、stage、summary 與 internal log reference。

## 5. Phase 4：Local Web App

目標：建立完整可操作的 local browser UI。

工作項目：

- 建立 React + TypeScript 前端。
- 建立 upload page，支援單檔、格式、大小、title 預檢。
- 建立 typed API client。
- 建立 job result page。
- 建立 job status polling。
- 顯示 stage、progress、warnings、errors。
- 顯示準確度提醒與輸出用途說明。
- 建立下載入口。

完成標準：

- 使用者可從瀏覽器完成上傳並進入 job page。
- 分析中能看到狀態更新。
- 完成後能下載輸出檔案。
- 分析失敗時顯示可理解錯誤與下一步建議。

## 6. Phase 5：Score Preview、Export 與 Artifact Management

目標：讓使用者能檢查輸出，並讓本機 artifacts 可管理。

工作項目：

- 整合 OpenSheetMusicDisplay。
- 建立 MusicXML preview component。
- 建立 PDF export 狀態與 fallback UI。
- 建立 artifact detail / log 檢視入口。
- 建立重新下載與重新執行 job 的規則。
- 設計本機 artifacts 清理策略。

完成標準：

- completed job 可在結果頁看到鼓譜預覽。
- MusicXML 可被 MuseScore 或相容軟體開啟。
- PDF 可下載並打開。
- PDF 失敗時，MIDI / MusicXML 仍可下載且 UI 顯示部分成功。

## 7. Phase 6：品質、可觀測性與可恢復性

目標：讓 V1 不只是能跑通，而是可診斷、可重現、可評估。

工作項目：

- 建立 unit tests、integration tests、pipeline fixture tests。
- 建立 frontend smoke / browser E2E。
- 建立人工準確度評估表。
- 建立 regression fixture set。
- 建立 structured pipeline logs。
- 建立 job crash / restart handling tests。
- 建立 V1 完整度 checklist。

完成標準：

- 固定 tests 可在本機通過。
- 人工評估流程可比較不同 pipeline 版本。
- 工程師可從 UI 或 log 找到失敗 stage 與主要原因。

## 8. Phase 7：Packaging、啟動流程與 V1 驗收

目標：讓一般技術使用者可重現啟動 local-first V1。

工作項目：

- 定義本機 app data 目錄與設定檔位置。
- 建立一條命令啟動 backend、frontend 與 job manager 的流程。
- 建立 runtime preflight check。
- 建立資料備份與重置文件。
- 建立 V1 release checklist。
- 文件化 future server mode / desktop shell 的邊界。

完成標準：

- 工程師可依 README 在乾淨本機啟動 GrooveScribe。
- `localhost` UI 可完成一首短音檔端到端流程。
- V1 完整度標準全部達成或明確列出例外。

## 9. 建議開發順序

```text
runtime baseline
→ SQLite schema + local storage
→ local job manager
→ pipeline service integration
→ upload/status/result/download API
→ local web upload UI
→ local web result UI
→ MusicXML preview + PDF fallback
→ artifact/log management
→ tests + manual eval
→ one-command local startup
→ V1 acceptance
```

## 10. 風險與優先處理

| 風險 | 優先處理方式 |
|---|---|
| Demucs / ADTOF 安裝困難 | 保留 runtime preflight、版本紀錄、mock mode 與清楚錯誤訊息 |
| 本機 CPU 執行太慢 | UI 顯示 progress、elapsed time 與預期耗時；文件標示 GPU 建議但不作 V1 必要條件 |
| SQLite job 狀態與長任務一致性 | job manager 採明確 transaction / checkpoint；重啟後標示 stale processing job |
| MIDI 準確度不足 | 保留 raw MIDI、processed MIDI、人工評估表與 pipeline version |
| PDF export 不穩 | MusicXML 作為主要譜面輸出；PDF 可標記部分失敗 |
| artifacts 佔用磁碟 | V1 設計清理策略與 artifact size summary |
| 未來 server mode 分歧 | V1 主線只做 local-first；Redis / Celery / PostgreSQL 作 adapter / optional mode |
