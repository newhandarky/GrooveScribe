# GrooveScribe V1 Release Notes

## Summary

GrooveScribe V1 is a local-first web app for turning a single MP3 or WAV file into a reviewable drum score draft on the user's machine. V1 uses a local FastAPI backend, React frontend, SQLite database, local job queue, and local filesystem artifacts.

This release candidate is validated through deterministic local gates, browser smoke tests, manual eval checks, artifact hygiene checks, redaction checks, release evidence generation, and RC handoff validation.

2026-07-07 RC finalization note: the default RC pilot handoff flow completed with `status=passed`, the handoff validator returned `issues=[]`, and the RC bundle redaction scan had no matches. true-AI remains opt-in and was not required for the deterministic sign-off path.

## Included Capabilities

- Local upload, status, result, download, job history, retry/rerun, and local data summary APIs.
- Localhost UI for upload, active job tracking, recent jobs, failed/interrupted recovery, MIDI/MusicXML downloads, MusicXML preview/fallback, PDF optional status, and review packet handoff.
- Runtime preflight for mock flow readiness and true-AI opt-in diagnostics.
- Pipeline quality diagnostics and artifact validation summaries for MIDI, MusicXML, and optional PDF.
- Review packet JSON/ZIP for completed jobs, including quality diagnostics, validation, reviewer checklist, and manual eval seed.
- Manual eval CSV gate and true-AI baseline tooling.
- Release gate, release evidence, local setup doctor, and RC pilot handoff bundle.

## Known Limitations

- true-AI is opt-in and is not a normal CI or tag blocker.
- PDF rendering is optional; unavailable or failed PDF export does not block MIDI/MusicXML handoff.
- V1 is not cloud/SaaS and does not include accounts, collaboration, sync, or hosted storage.
- V1 does not include a Tauri/Electron shell.
- V1 produces a reviewable and manually correctable drum score draft; it does not claim publication-grade transcription accuracy.
- Storage cleanup and reset remain dry-run only; V1 does not provide destructive deletion flows.

## Artifact Hygiene

Do not commit generated local outputs:

- `frontend/dist`
- `storage/`, `backend/storage/`, `worker/storage/`
- SQLite/DB files
- tmp artifacts
- Playwright reports: `playwright-report/`, `test-results/`, `blob-report/`
- generated release evidence
- generated review packets
- generated RC handoff bundles

Generated release artifacts should stay outside the repo and be regenerated by the documented commands.
