# GrooveScribe 資料模型設計

## 1. 設計原則

- V1 預設使用 SQLite。
- 大型檔案不存 database，只存 storage key 與 metadata。
- `TranscriptionJob` 是主要流程狀態來源。
- 每個輸出檔案都建立 `ExportFile` 紀錄。
- AI 模型名稱、版本、pipeline config 需保留，方便比較準確度。
- V1 不要求登入；`User` 為 future optional。

## 2. AppSettings / RuntimeSnapshot

用途：記錄本機 app data、runtime preflight 與版本資訊。

欄位建議：

| 欄位 | 型別 | 說明 |
|---|---|---|
| id | string | 設定 key |
| value | json / text | 設定值 |
| updated_at | datetime | 更新時間 |

Runtime snapshot 可記錄：

- Python version
- ffmpeg version
- Demucs version
- ADTOF command template / version
- MuseScore renderer path / version
- pipeline version

## 3. User

用途：future cloud sync / server mode 使用者帳號與歷史紀錄。V1 可不建立使用者，或允許 `user_id` 為 null。

欄位建議：

| 欄位 | 型別 | 說明 |
|---|---|---|
| id | uuid / string | 使用者 ID |
| email | string nullable | Email |
| display_name | string nullable | 顯示名稱 |
| created_at | datetime | 建立時間 |
| updated_at | datetime | 更新時間 |

## 4. AudioFile

用途：記錄使用者上傳的原始音檔與標準化音檔 metadata。

欄位建議：

| 欄位 | 型別 | 說明 |
|---|---|---|
| id | uuid / string | 音檔 ID |
| user_id | uuid nullable | Future optional 使用者 |
| original_filename | string | 原始檔名 |
| content_type | string | MIME type |
| file_size_bytes | bigint | 原始檔案大小 |
| duration_seconds | float nullable | 音檔長度 |
| sample_rate | int nullable | 原始或標準化 sample rate |
| channels | int nullable | 聲道數 |
| original_storage_key | string | 原始檔 storage key |
| normalized_storage_key | string nullable | 標準化 WAV storage key |
| created_at | datetime | 建立時間 |

## 5. TranscriptionJob

用途：代表一次音訊轉鼓譜任務。

欄位建議：

| 欄位 | 型別 | 說明 |
|---|---|---|
| id | uuid / string | Job ID |
| user_id | uuid nullable | Future optional 使用者 |
| audio_file_id | uuid | 對應 AudioFile |
| status | enum | job 狀態 |
| stage | enum | 目前 pipeline stage |
| progress | int | 0-100 |
| title | string nullable | 顯示標題 |
| pipeline_version | string | pipeline 版本 |
| source_separator | string nullable | 例如 `demucs` |
| source_separator_version | string nullable | 模型或套件版本 |
| drum_transcriber | string nullable | 例如 `adtof-pytorch` |
| drum_transcriber_version | string nullable | 模型或套件版本 |
| runtime_snapshot_id | string nullable | runtime 記錄 reference |
| error_code | string nullable | 錯誤碼 |
| error_message | text nullable | 使用者可理解錯誤 |
| error_stage | string nullable | 發生錯誤的 stage |
| internal_error_ref | string nullable | log reference |
| queued_at | datetime nullable | 入 queue 時間 |
| started_at | datetime nullable | 開始處理時間 |
| completed_at | datetime nullable | 完成時間 |
| failed_at | datetime nullable | 失敗時間 |
| interrupted_at | datetime nullable | service restart / interrupted 時間 |
| created_at | datetime | 建立時間 |
| updated_at | datetime | 更新時間 |

## 6. DrumTrack

用途：記錄轉譜結果的摘要與 artifacts。

欄位建議：

| 欄位 | 型別 | 說明 |
|---|---|---|
| id | uuid / string | DrumTrack ID |
| job_id | uuid | 對應 TranscriptionJob |
| drums_stem_storage_key | string nullable | Demucs drums stem |
| raw_midi_storage_key | string nullable | ADTOF 原始 MIDI |
| processed_midi_storage_key | string nullable | 後處理 MIDI |
| drum_events_storage_key | string nullable | 中間事件 JSON |
| estimated_bpm | float nullable | 估計 BPM |
| time_signature | string | V1 預設 `4/4` |
| event_count | int | drum events 數量 |
| confidence_label | enum nullable | low / medium / high |
| warnings | json | 警告代碼陣列 |
| created_at | datetime | 建立時間 |

## 7. ExportFile

用途：記錄可下載輸出檔。

欄位建議：

| 欄位 | 型別 | 說明 |
|---|---|---|
| id | uuid / string | ExportFile ID |
| job_id | uuid | 對應 TranscriptionJob |
| type | enum | midi / musicxml / pdf |
| status | enum | available / failed |
| storage_key | string | 檔案 storage key |
| content_type | string | MIME type |
| file_size_bytes | bigint nullable | 檔案大小 |
| checksum | string nullable | 檔案校驗 |
| error_code | string nullable | 產生失敗錯誤碼 |
| created_at | datetime | 建立時間 |

## 8. JobStatus

建議 enum：

```text
uploaded
queued
processing
completed
failed
interrupted
canceled
```

V1 可以先不實作 cancel UI，但 schema 可保留 `canceled`。

## 9. Stage Enum

建議 enum：

```text
uploaded
queued
preprocessing
source_separation
stem_validation
drum_transcription
midi_post_processing
notation_generation
pdf_export
completed
failed
```

## 10. 狀態流轉設計

允許流轉：

```text
uploaded → queued
queued → processing
processing → completed
processing → failed
processing → interrupted
interrupted → queued
queued → failed
```

Stage 流轉：

```text
preprocessing
→ source_separation
→ stem_validation
→ drum_transcription
→ midi_post_processing
→ notation_generation
→ pdf_export
→ completed
```

規則：

- `completed_at` 只能在 status 進入 completed 時設定。
- `failed_at` 只能在 status 進入 failed 時設定。
- failed job 必須有 `error_code`、`error_message`、`error_stage`。
- interrupted job 必須能被 UI 顯示，並提供重新執行或標記失敗策略。
- job manager 更新 progress 時需以 job id 查詢，避免信任 queue payload 中的所有資料。

## 11. Artifact Storage Key 建議

```text
jobs/{job_id}/original/{filename}
jobs/{job_id}/audio/normalized.wav
jobs/{job_id}/stems/drums.wav
jobs/{job_id}/midi/raw_drum.mid
jobs/{job_id}/midi/processed_drum.mid
jobs/{job_id}/midi/drum_events.json
jobs/{job_id}/notation/score.musicxml
jobs/{job_id}/exports/score.pdf
jobs/{job_id}/logs/pipeline.json
```

這種 key 命名可直接映射到 local filesystem，也可在 future server mode 映射到 S3 object key。
