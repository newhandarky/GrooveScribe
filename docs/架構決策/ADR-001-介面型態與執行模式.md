# ADR-001：介面型態與執行模式

## Status

Accepted

## Date

2026-07-01

## Context

GrooveScribe 需要執行 ffmpeg、Demucs、ADTOF-pytorch、MusicXML generation 與 PDF export。這些工作具有以下特性：

- AI runtime 安裝與硬體支援差異大。
- 單首音檔處理時間可能從數十秒到數分鐘以上。
- 輸入音檔與中間 artifacts 可能較大。
- 使用者需要能檢查 raw MIDI、processed MIDI、MusicXML、PDF 與 pipeline log。
- 第一版目標是完整 V1 產品，不是只跑通流程或快速雲端 demo。

可選介面與執行模式：

1. 雲端 Web App：前端與 API 部署在 server，worker 在 server 或 GPU worker 上執行。
2. Desktop App：使用 Tauri / Electron 封裝 UI 與本機服務。
3. Local Web App：使用者本機啟動 backend / frontend / job manager，透過瀏覽器操作 localhost。

## Decision

GrooveScribe V1 選擇 Local Web App 作為預設介面與執行模式。

V1 預設組成：

- Browser UI on localhost。
- React + TypeScript frontend。
- 本機 FastAPI backend。
- SQLite database。
- 本機 job manager。
- 本機 filesystem artifacts。
- 本機 Python AI runtime。

V1 不把以下項目作為預設主線：

- 雲端部署。
- SaaS 帳號與多租戶。
- Redis / Celery queue。
- PostgreSQL database。
- S3-compatible storage。
- Tauri / Electron desktop shell。

上述項目保留為 future / optional capability。

## Rationale

### 為什麼不是雲端 Web App

- AI runtime 與 GPU / CPU 環境成本高，早期雲端部署會把產品問題與 infra 問題綁在一起。
- 使用者音檔可能有版權、隱私與上傳限制，本機處理較符合 early product trust。
- V1 的核心風險是轉譜品質、pipeline 穩定性與 artifact 可診斷性，不是多租戶部署。
- 雲端需要帳號、storage policy、queue scaling、GPU worker、成本控制與資料清理，會稀釋 V1 主線。

### 為什麼不是先做 Tauri / Electron

- Desktop shell 會引入額外 packaging、auto-update、native permission、跨平台啟動與 runtime bundle 問題。
- Local Web App 已能提供完整操作介面，並保留瀏覽器開發與除錯效率。
- 先穩定 backend、pipeline、job manager、storage 與 UI workflow，再封裝 shell 風險較低。

### 為什麼選 Local Web App

- 使用者不需要把音檔上傳到雲端。
- 開發團隊可以先專注於 pipeline 品質、job state、artifacts 與本機 UX。
- SQLite + local filesystem 降低安裝依賴。
- Browser UI 仍能使用成熟的 React / TypeScript / OpenSheetMusicDisplay 生態。
- 未來可以在相同核心架構上增加 desktop shell 或 server mode。

## Consequences

### 正面影響

- V1 安裝與執行依賴更少。
- 音檔與 artifacts 預設留在使用者本機。
- Pipeline logs、fixtures、manual eval 與 regression workflow 更容易對齊本機環境。
- UI 與 backend 邊界仍清楚，未來可封裝桌面或部署 server。

### 代價

- 使用者需要啟動本機服務，V1 需要清楚的啟動與 preflight UX。
- 本機硬體性能會影響處理時間。
- 跨平台 runtime 安裝仍需處理。
- 多裝置同步、分享、歷史雲端保存不在 V1 內。

## Implementation Notes

- 文件與 ticket 應以 SQLite、本機 job manager、local filesystem 為預設。
- Redis / Celery / PostgreSQL / S3 只能出現在 future optional 或 server mode 章節。
- `docker-compose.yml` 可保留為既有 server-style 開發輔助，但不可被描述為 V1 必要路徑。
- 若未來加入 Tauri / Electron，shell 應包裝既有 Local Web App 與本機 service，不應重寫核心 pipeline。

## Review Triggers

重新評估此 ADR 的條件：

- V1 local-first 完整度標準已達成，且多數使用者需要一鍵桌面封裝。
- 產品需要跨裝置同步或分享。
- 需要多人協作或 hosted processing。
- 本機 AI runtime 安裝成本高到明顯阻礙使用。
